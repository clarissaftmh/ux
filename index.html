What is Exploration?

Exploration is OCBC NISP's higher education venture that helps high school students in getting to their dream major and university. They provide solution for students to find and plan their higher education. Greatly collaborate with various University and School to provide classes, verified information, and other related activities that help students achieve their ideal university and major

🤔 A bit of back story....

Exploration previous released on explore major and campus were still only capturing the bigger picture. However, they were aimed to make it more unique by adapting the current character identity called “Serra”. Me and my team set the the goal to enhance its experience in explore, detail, and comparison pages with combining the “Serra” character identity.

Main focus

The main focus during the revamp was the Explore Campus Page, Kampus Detail Page, Explore Jurusan Page, Jurusan Detail Page, Bandingkan Kampus.

Process Followed:

Client brief → Competitor analysis (quipper, bibit, blibli) → Analyze user/competitor insight → Sitemap and user journey decision making → WIP → Wireframing → WIP → Visualization high fidelity → WIP → Final Deliverables.

Competitor Analysis

We did a competitor analysis in order to find the best practice in comparison product and explore card.

Some interesting data points :

Based on the competitor approach, we can gather some insightful data points are :

  1. From bibit, we can capture the ideal journey for “comparing product” which can be implemented to our “bandingkan kampus” page. Adding 3 products are quite ideal for our decision making.

  2. But, at the second screen, it seems like a categories, so the user tend to have one more clicks to go in product card (this is not recommended if the user do not know which category they must in), I supposed there must be some filter instead of page categories.

  1. Blibli created the comparison journey seems intuitively, but somehow the entry points feature was unseen if the user scan the page only in short of time.

  2. The third page was greatly captured as “comparing while exploring” this journey well match with our purpose on comparing product.

  1. When we first jump on this page, the card pretty similar with exploration card on campus and major. Interesting point that we gather was the elements ordering, start with CTA personality test, Search bar, Explore Card Kampus.

🤔 A bit of back story....

Main focus

The main focus was the creative discovery on personality take quiz and result.

Process Followed:

Client brief → Creative discovery → Sitemap and user journey decision making → WIP -> High fidelity → WIP → Iteration design

Creative discovery

At this point, we accelerate the character identity into 3 keywords: smart, sharp, and passionate.


Thinking about journey

the journey come up with converting the entry points might be ideal for the feature. When the user jump into personality quiz, then will entering the feature as"Konsultasi Minat Bakat session".

Ordering the sitemap

It was the overall sitemap for implementing the Serra character, but I will deeply explain the "Karatermu" page only since it was my scope of work.

Karaktermu page can be accessed if the user already take the personality quiz because it was part of the result test. The test will generate the top 3 characters which each character have it's own uniqueness. WHY 3? because 3 option was the ideal version for the user and reduce confusion.
